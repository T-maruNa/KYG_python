# AI Virtual Investment Battle

このプロジェクトは、AIキャラクターによる仮想投資バトルを自動運用し、WordPressブログへ **営業日の朝8時と夜22時の2記事** を自動投稿するためのPythonバッチです。

## セットアップ

```bash
# 1. 依存パッケージのインストール
pip install -r batch/requirements.txt

# 2. 環境変数ファイルの作成
cp batch/.env.example batch/.env
# batch/.env を編集して各キーを入力する

# 3. DBの初期化
cd batch
python -c "from src.database.db_initializer import DBInitializer; DBInitializer().initialize()"

# 4. 朝記事の生成（ドライラン）
python main.py --mode morning --dry-run

# 5. 夜記事の生成（ドライラン）
python main.py --mode night --dry-run
```

## AIモデル構成

初期運用では **OpenAI に一本化** しています。

| 用途 | プロバイダー | モデル（デフォルト） |
|---|---|---|
| テキスト生成（全キャラ・記事生成） | OpenAI | `gpt-4.1-mini` |
| 画像生成 | OpenAI Images API | `gpt-image-1` |

キャラクターごとにAIモデルを分ける構成は廃止しました。キャラクター差はモデルの違いではなく、以下のプロンプトで出します。

- `batch/prompts/character_profile.md`
- `batch/prompts/common_rules.md`
- `batch/prompts/prediction_article_prompt.md`
- `batch/prompts/result_article_prompt.md`
- `batch/prompts/talk_prompt.md`

Google Gemini は初期運用では使いません。将来のマルチプロバイダー実験用にコードと環境変数の枠だけ残しています（`ENABLE_GOOGLE_PROVIDER=false`）。

## 企画概要

3人のAIキャラクターが、それぞれ月初に仮想資金100万円を受け取ってスタートします。

毎営業日、各キャラクターは以下の3価格帯から1銘柄ずつ選び、最大3銘柄へ仮想投資します。

- 小型株: 100円以下
- 中型株: 100円超から1000円以下
- 大型株: 1000円超から10000円以下

翌営業日に結果を検証し、キャラクター別の資産推移、勝敗、価格帯別成績をブログ記事として公開します。

月の最終営業日には、その月の最終結果を月間まとめ記事として公開します。

- `rei`

  - 画像イメージ: 朝のデスクでノートとPCを開く、落ち着いたお姉さん

- `mirai`

  - 企業イメージ、テーマ性、人気度を重視
  - 画像イメージ: リクルートスーツ姿の小柄な新社会人、手帳や作戦メモを持ちながら前向きに頑張る

- `ritu`
  - ランダム、直感枠
  - AI予測がランダムに勝てているかを見るためのベースライン
  - 画像イメージ: 金髪ギャル、ネイル、アクセサリー、サイコロ風アイテムで盛り上げるアゲアゲ系のお姉さん

キャラクター画像は、投資システム感よりも女の子たちの日常ブログ感を優先します。

- 成人女性として描く
- オリジナルキャラクターにする
- ブログのプロフィール、記事ヘッダー、会話風コメントに使える絵柄にする
- 銘柄コード、株価、投資助言文、ロゴ、ブランド名は画像内に入れない
- 毎朝の企画連載に親しみが出る、生活感のある雰囲気にする

- `assets/characters/rei/base.png`
- `assets/characters/mirai/base.png`
- `assets/characters/ritu/base.png`

## 仮想売買ルール

- 月初資金: 各キャラクター100万円
- 投資枠: 小型、中型、大型の3枠
- 投資額: 小型株は30万円まで、中型株は30万円まで、大型株は40万円まで
- 予備現金: 基本は0円。ただし資金残高によって投資枠を制限する
- 買値: 予測材料に使った前営業日の終値
- 売値: 次の営業日の終値
- 保有期間: 1営業日
- 株数: 各価格帯の投資上限額以内で買える整数株
- 手数料: MVPではなし
- 持ち越し: なし
- 損益: `(売値 - 買値) * 株数`
- 損益率: `(売値 - 買値) / 買値 * 100`
- 勝敗: 売値が買値より高ければ勝ち
- 月次リセット: 月初の最初の営業日に各キャラクターの仮想資金を100万円に戻す
- 月間成績: 月内の日次損益を積み上げ、最終営業日に月間ランキングを確定する

資金残高による投資制限は以下とします。

- 資金残高が60万円以上: 小型、中型、大型の3枠すべてに投資する
- 資金残高が30万円以上60万円未満: 大型株には投資せず、小型と中型の2枠に投資する
- 資金残高が30万円未満: 小型株のみ投資する

投資可能額は、各枠の上限と現在資金残高の範囲内で決めます。

日足OHLCでは高値と安値の到達順が分からないため、損切りや利確ルールはMVPでは入れません。

## 月次運用ルール

この企画は月ごとのリーグ戦として運用します。

- 月初の最初の営業日に、各キャラクターへ100万円を配布する
- 月中は日次損益を資産に反映する
- 資金が減ったキャラクターは、残高に応じて投資できる価格帯が制限される
- 月をまたいで資産は引き継がない
- 月の最終営業日に、月間成績を確定する
- 翌月は全キャラクター100万円から再スタートする

月間まとめでは、以下を発表します。

- 月間総合ランキング
- キャラクター別の最終資産
- キャラクター別の月間損益
- 価格帯別の勝率
- 一番大きく勝った銘柄
- 一番大きく負けた銘柄
- 月間MVPキャラクター
- 先月までの累計MVPキャラクター
- 累計成績ランキング
- 翌月へのコメント

月ごとの資金はリセットしますが、過去月の成績は累計記録として残します。

累計記録では、以下を集計します。

- 累計MVP回数
- 累計月間優勝回数
- 累計損益
- 月平均損益
- 累計勝率
- 価格帯別の累計勝率
- 最大月間利益
- 最大月間損失

## 記事構成

記事は投資レポートではなく、「株をする3人の女の子たちの日常ブログ・連載企画」として読めるように構成します。

### 朝記事（朝8時：作戦会議）

3人が今日の銘柄選択と作戦を発表する記事です。

1. **冒頭リード** — 今日の3人の空気を1〜2文で表現するAI生成テキスト
2. **今朝の順位メモ** — 現時点の月間ランキングをコンパクトに表示
3. **今日の作戦会議** — 3人の掛け合いセリフ（AI生成）
4. **今日の作戦ノート** — 各キャラの銘柄選択カード＋作戦コメント
5. **今朝の3人** — 月間順位・他2人の選択への反応を踏まえたセリフ（AI生成）
6. **夜22時への導線** — 結果発表予告テキスト
7. **免責文**

### 夜記事（夜22時：結果発表）

その日の勝負結果をキャラクターのドラマとして届ける記事です。

1. **冒頭リード** — 今日の勝負がどういう一日だったかをキャラクター主語で書くAI生成テキスト
2. **今日の主役** — 最も絶対値で損益が大きかったキャラクターをフィーチャー
3. **今日の勝負結果** — キャラクター別の損益率と一言コメント
4. **今日の反省会** — 3人が今日の結果について比較しながら振り返るAI生成会話
5. **今日の名場面** — その日のキャラクターの名場面を1文ずつ切り取ったAI生成テキスト
6. **今月のランキング** — 現在の資産ランキング（キャラクター画像付き）
7. **次回へのひとこと** — 翌日への予告・煽りテキスト（AI生成）
8. **累計成績** — 過去月を含めた累計損益・MVP・優勝回数
9. **免責文**

月の最終営業日には、通常の日次記事に加えて月間まとめ記事も生成します。

月の最終営業日には、通常の日次記事に加えて月間まとめ記事も生成します。

土日祝、休場日は通常の勝負記事を投稿しません。

ただし、MVP後の拡張として以下の記事を別枠で生成できます。

- 土曜: 週次まとめ、今週の勝者、反省会
- 日曜: キャラクターの日常ブログ、来週の作戦会議
- 祝日、休場日: 市場がお休みのお知らせ、直近ランキング、次回予告

記事では、日次、月次、累計の金額結果を表示します。

- 日次結果: 当日の損益額、当日終了時点の資産額
- 月次結果: 月初100万円からの増減額、月末または当日時点の月間資産額
- 累計結果: 過去月を含めた累計損益額、累計MVP回数、累計優勝回数

ブログ本文には、yfinance由来のOHLC生データを大量掲載しません。

公開する情報は、主に以下に絞ります。

- 銘柄コード
- 銘柄名
- AIの予想理由
- 自分の考察、またはキャラクターコメント
- 検証結果の要約
- 的中、外れ
- 損益率などの要約値

## 免責方針

記事は投資助言ではなく、AIキャラクターによる仮想投資シミュレーション企画として扱います。

記事末尾には、毎回以下の趣旨の免責文を入れます。

```text
本記事はAIキャラクターによる仮想投資シミュレーションです。
実際の売買を推奨するものではありません。
投資判断はご自身の責任で行ってください。
```

以下のような表現は避けます。

- 絶対上がる
- 必ず儲かる
- 買うべき
- 推奨銘柄
- 利益保証
- 確実

## データ取得方針

株価データはyfinanceで取得します。

ただし、yfinanceの日本株日足データは更新時刻が保証されないため、バッチでは以下の方針にします。

- 朝記事（8時）に間に合うよう、早朝にデータ取得を試す
- 対象日のデータが存在するか確認する
- データ未反映ならリトライする
- それでも取れない場合は記事投稿を止める、または下書き扱いにする
- 祝日、休場日は最新の取得可能営業日を使う

## AIに渡す材料

現状は前営業日1日分のOHLCと出来高だけをAIに渡しています。

精度改善のため、次の特徴量を追加します。

- 前日比
- 5日騰落率
- 20日騰落率
- 出来高変化率
- 5日移動平均
- 20日移動平均
- ボラティリティ
- 直近高値更新
- 直近安値更新

AIには、各銘柄の生OHLCだけでなく、上記特徴量をCSVとして渡します。

## WordPress投稿方針

WordPress REST APIを使って自動投稿します。

- 投稿時刻: 朝記事は営業日の朝8時、夜記事は営業日の夜22時
- 投稿形式: `status=future` による予約投稿
- 認証情報: `.env` から読み込む
- 二重投稿: 同一日付の記事がある場合は投稿しない
- 土日祝、休場日は通常の勝負記事を投稿しない

投稿前には機械的なチェックを行います。

- 3キャラ x 3価格帯 = 9件のエントリーがあること
- 銘柄コードが空でないこと
- 銘柄名が空でないこと
- 免責文が含まれていること
- 禁止表現が含まれていないこと
- 記事本文が空でないこと

## ブログ記事のDB保存方針

生成した記事本文はWordPressへ投稿するだけでなく、DBにも保存します。

保存先テーブル: `t_blog_post_history`

保存する内容:

- 記事日付
- 記事種別（`prediction_daily` / `result_daily` / `monthly`）
- タイトル
- 本文（HTML）
- WordPressのpost ID
- 投稿ステータス（`scheduled` / `dry_run` / `failed` / `skipped`）

DBに本文を保持する理由:

- WordPressに依存しない形でコンテンツを再利用できる
- 将来的にDBだけからサイトを構築できる
- 投稿失敗時に本文を失わずリトライできる
- 記事内容の履歴として参照できる

## AI API予算ガード

AI API代が想定外に膨らまないよう、実装では予算ガードを入れます。

目標は、通常運用で月数百円から数千円程度に抑えることです。

月数万円規模になりやすい原因は、以下のような処理です。

- 全銘柄の長い株価履歴をそのままAIへ渡す
- 画像を毎日何枚も生成する
- 記事本文を失敗時に何度も再生成する
- API失敗時に無制限リトライする
- AIへ渡す候補銘柄数を制限しない

そのため、以下の制限を設定値として持ちます。

```text
MONTHLY_AI_BUDGET_JPY=3000
DAILY_AI_CALL_LIMIT=10
AI_RETRY_LIMIT=2
ENABLE_DAILY_IMAGE_GENERATION=false
ENABLE_MONTHLY_MVP_IMAGE=true
MAX_CANDIDATES_PER_RANGE=30
```

各設定の意味は以下です。

- `MONTHLY_AI_BUDGET_JPY`
  - 月額AI API予算の目安
  - この金額を超えそうな場合は、AI呼び出しや画像生成を止める
- `DAILY_AI_CALL_LIMIT`
  - 1日に実行できるAI呼び出し回数の上限
  - 予測（アナリスト分）＋記事リード＋今日の主役コメント＋反省会会話＋次回予告＋キャラクターコメント分を含めて数える
  - 現在の実装では1日あたり最大13回程度のAI呼び出しが発生するため、推奨値は10
- `AI_RETRY_LIMIT`
  - AI/API失敗時の最大リトライ回数
  - 無限リトライは禁止
- `ENABLE_DAILY_IMAGE_GENERATION`
  - 日次記事ごとの画像生成を有効にするか
  - MVPでは `false` とし、固定画像や表情差分を使う
- `ENABLE_MONTHLY_MVP_IMAGE`
  - 月間MVPなど特別記事用の画像生成を有効にするか
- `MAX_CANDIDATES_PER_RANGE`
  - AIへ渡す候補銘柄数の上限
  - 小型、中型、大型それぞれ最大30銘柄程度に絞る

通常運用では、以下の方針にします。

- 毎営業日の画像生成はしない
- キャラクター画像は固定画像または表情差分から選ぶ
- AIへ渡す株価情報はPython側で特徴量に変換して要約する
- AIへ渡す候補銘柄は価格帯ごとに絞る
- ブログ本文生成は原則1日1回
- リトライは最大2回まで
- 予算上限に近づいた場合は、記事生成をテンプレート寄りに切り替える

## 修正順

### 1. 文字化けと命名の整理

- コメント、ログ、プロンプトの文字化けを直す
- 既存の仕様名を整理する
- 既存の処理意図を読みやすくする

### 2. DBテーブル追加

仮想投資バトル用のテーブルを追加します。

- キャラクター資産テーブル
- 投資履歴テーブル
- 日次成績テーブル
- 月次成績テーブル
- 必要に応じてブログ投稿履歴テーブル

既存の `m_stock`, `t_stock_actual`, `t_stock_predict` はできるだけ活かします。

### 3. 過去データ取得

- yfinance取得を過去20営業日以上に対応する
- 重複登録を避ける
- 休場日、祝日、データ未反映に対応する
- 取得済みの最新営業日を判断できるようにする

### 4. 特徴量作成

- 過去データから特徴量を計算する
- AI入力用のCSVを生成する
- 価格帯ごとに候補銘柄を分ける
- キャラクターの資金残高に応じて、AIへ依頼する価格帯を変える

### 5. AI予測入力の改善

- 小型、中型、大型から1銘柄ずつ選ばせる
- 資金残高が60万円未満なら大型を選ばせない
- 資金残高が30万円未満なら小型だけを選ばせる
- 出力形式を固定する
- キャラクターごとの性格を明確にする
- 過去成績をプロンプトへ入れられる構造にする

### 6. 仮想売買と結果検証

- 月初の最初の営業日に各キャラクターへ100万円を設定する
- 資金残高に応じて投資対象の価格帯を制限する
- 前営業日終値で買ったことにする
- 次営業日終値で売ったことにする
- 株数、購入額、売却額、損益、損益率を保存する
- キャラクター別、価格帯別に勝敗を保存する

### 7. 成績集計

- キャラクター別の現在資産
- キャラクター別の日次損益額
- キャラクター別の累計損益
- 月間ランキング
- 月間最終資産
- 月初100万円からの月次増減額
- 累計MVP
- 累計月間優勝回数
- 累計損益
- 価格帯別の勝率
- 直近N回の勝率
- `ritu` との比較

### 8. ブログ本文生成

- 前回結果
- 日次の金額結果
- 月次の金額結果
- 累計の金額結果
- 現在ランキング
- 今日のエントリー銘柄
- キャラクターコメント
- 結果に応じた表情画像
- 検証コメント
- 免責文

を含む記事本文を生成します。

月の最終営業日には、月間成績まとめ記事も生成します。

月間まとめ記事には、先月までの累計MVPや累計ランキングも含めます。

- WordPress REST APIクライアントを追加する
- `.env` にWordPress設定を追加する
- 同一日付の二重投稿を防ぐ
- 朝8時・夜22時の2本立て予約投稿に対応する

### 10. 運用安定化

- ログ出力
- API失敗時のリトライ
- AI API予算ガード
- 日次AI呼び出し回数の制限
- 月額予算上限の制御
